# Case Model — Systemic Homeopathy

This file defines a standard way to analyze and describe cases based on the Life Force manual.

---

## 1. Core Idea

A case is not only about symptoms.

A case shows:

- initial state
- archetypal pattern
- remedy selection
- state transformation
- dynamic change of remedies

---

## 2. Case Structure

```md
# Case [Name]

## Request

## Initial State

Describe the state and use an archetypal image.

## Hidden Mechanism

What drives the symptom?

## First Remedy Set

- remedy 1 — why
- remedy 2 — why

## First Change

What changed after first cycle?

## Second State (Archetype)

## Second Remedy Set

## Second Change

## Third State

## Third Remedy Set

## Final State

## Observed Transformation

Describe how the person changed:

- energy
- behavior
- relationships
- business or life situation

## Key Insight

What does this case show about the method?
```

---

## 3. Archetypal Tracking

Track state through images:

- sheep → wolf → bear → tiger
- broken → supported → stable → active → confident

The shift of archetypes shows internal transformation.

---

## 4. Remedy Dynamics

Important principle:

Remedies change as the state changes.

Track:

- which remedy worked
- how long it worked
- when it stopped working
- what new remedy became relevant

---

## 5. Time Cycles

Typical rhythm:

- 1 week — first observable change
- 2–4 weeks — deeper transformation

---

## 6. What To Extract From Cases

Each case should show:

- pattern of change
- sequence of remedies
- state transformation
- how accurately the method worked

---

## 7. Main Rule

A case is proof of method in action.

It should show:

state → remedy → change → next state → new remedy → transformation
